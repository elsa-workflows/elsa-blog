<section><div><div><h3>What’s new in Elsa 2.0</h3><p>The Elsa 2.0 release is out, and it’s a big one!</p></div><div><figure><img data-image-id="1*diMew05OXAqAM7xF3T3wVQ.png" data-width="3734" data-height="805" data-is-featured="true" src="https://cdn-images-1.medium.com/max/2560/1*diMew05OXAqAM7xF3T3wVQ.png"></figure></div><div><p>This is the second release since Elsa 1.0 end of November 2019 and it contains many improvements and new features.</p><p>Let’s take a look.</p><h3>What’s New</h3><ol><li>Brand new designer with pan &amp; zoom support and auto-layout</li><li>Improved Workflow Builder API for Coded Workflows</li><li>Coded Workflows visualization thanks to auto-layout</li><li>Distributed Server / Multi Node support</li><li>Elsa Server API Endpoints</li><li>Clean separation between Elsa Server and Elsa Dashboard</li><li>Composite Activities</li><li>Activity Editors with Intellisense support powered by <a href="https://microsoft.github.io/monaco-editor/" rel="noopener" target="_blank">Monaco Editor</a></li><li>Workflow Context Support</li><li>New activities</li></ol><h4>1. Brand new designer with pan &amp; zoom support and auto-layout</h4><p>The designer has been completely rewritten from scratch, replacing <a href="https://jsplumbtoolkit.com/" rel="noopener" target="_blank">JsPlumb </a>with <a href="https://github.com/dagrejs/dagre-d3" rel="noopener" target="_blank">Dagre-D3</a> which offers better performance, easier to use API and built-in pan &amp; zoom support.</p><figure><img data-image-id="1*FJOGNRmGQSL_cYlf9oVOjw.gif" data-width="1742" data-height="1341" src="https://cdn-images-1.medium.com/max/800/1*FJOGNRmGQSL_cYlf9oVOjw.gif"><figcaption>Elsa’s new designer supports pan &amp; zoom</figcaption></figure><h4>2. Improved Workflow Builder API for Coded Workflows</h4><p>The workflow builder API received significant improvements, the most important one being that you can now use C# lambdas for workflow expressions, rather than having to use JavaScript for dynamic expressions.</p><p>In addition, many activities now come with extension methods that make it even easier to write workflow classes.</p><p>For example, in <strong>Elsa 1</strong> the HelloWorld workflow might look like this:</p><pre>builder.StartWith&lt;WriteLine&gt;(x =&gt; x.TextExpression = new LiteralExpression(&quot;Hello World!&quot;))</pre><p>In <strong>Elsa 2</strong>, the same workflow looks like this:</p><pre>builder.WriteLine(&quot;Hello World!&quot;);</pre><p>And to write someone’s name, the workflow might look like this:</p><pre>builder<br>   .WriteLine(&quot;Please enter you name:&quot;)<br>   .ReadLine()<br>   .WriteLine(context =&gt; $&quot;Nice to meet you, {context.GetInput&lt;string&gt;()}!&quot;);</pre><p>Notice that the second write line activity uses a C# lambda statement that dynamically sets the text to write based on the received input. In Elsa 1, you would have to resort to a JavaScript or Liquid expression. Which you can still do of course, and is some cases even more convenient such as when generating longer templated strings using Liquid syntax.</p><h4>3. Coded Workflows visualization thanks to auto-layout</h4><p>Thanks to the new designer implementation, it is now possible to display your coded workflows in the Elsa Dashboard.</p><p>For example, take the following workflow:</p><pre>public class HeartbeatWorkflow : IWorkflow<br>{<br>    private readonly IClock _clock;<br>    public HeartbeatWorkflow(IClock clock) =&gt; _clock = clock;<br><br>    public void Build(IWorkflowBuilder builder)<br>    {<br>        builder<br>            .WithDisplayName(&quot;Timer&quot;)<br>            .Timer(Duration.FromSeconds(30))<br>            .WriteLine(() =&gt; $&quot;Heartbeat at {_clock.GetCurrentInstant()}&quot;);<br>    }<br>}</pre><p>When you go to the Workflow Registry screen in the Elsa Dashboard, you will see this:</p><figure><img data-image-id="1*3r7a5iMwBfAOUKctJIChSw.png" data-width="1612" data-height="1565" src="https://cdn-images-1.medium.com/max/800/1*3r7a5iMwBfAOUKctJIChSw.png"><figcaption>Coded workflows can now be visualized</figcaption></figure><h4>4. Distributed Server / Multi Node support</h4><p>A critical limitation with Elsa 1 was that it did not operate well in multi-node scenarios. Background timers would cause workflows to execute on all nodes, and there was no distributed locking in place to prevent concurrency conflicts when two nodes would update the same workflow instance.</p><p>In Elsa 2, workflows now operate smoothly across multiple nodes thanks to distributed locking &amp; signaling.</p><p>You can learn more about this feature <a href="https://elsa-workflows.github.io/elsa-core/docs/next/hosting/hosting-distributed-hosting" rel="noopener" target="_blank">here</a>.</p><h4>5. Elsa Server API Endpoints</h4><p>Elsa 2 exposes REST API endpoints that enable easy integration with external applications. It contains endpoints to manage workflow definitions as well as to trigger and signal workflows.</p><h4>6. Clean separation between Elsa Server and Elsa Dashboard</h4><p>The Elsa Server API Endpoints mentioned above allows for a clean separation between the Elsa Dashboard and Elsa Server.</p><p>This is in contrast to the Elsa 1 Dashboard MVC Area package where you had to register activities both with the server as well as the dashboard.</p><p>With Elsa 2, you register the available activities in the Startup class of your server app, and the Elsa Dashboard, which by the way is a simple SPA built with Stencil, will interact with the Elsa Server API to query the available activities.</p><h4>7. Composite Activities</h4><p>Composite Activities are a new class of custom activities that you can write using the Workflow Builder API.</p><p>Example:</p><pre>public class AskNameActivity : CompositeActivity<br>{<br>    public override void Build(ICompositeActivityBuilder builder)<br>    {<br>        builder<br>            .WriteLine(&quot;Hi there! Please tell me your name.&quot;)<br>            .ReadLine()<br>            .Then&lt;Finish&gt;(finish =&gt; finish.WithOutput(context =&gt;context.GetInput&lt;string&gt;()));<br>    }<br>}</pre><p>These activities are like mini-workflows that you can add to your workflow.</p><blockquote>A future update will enable you to create Composite Activities straight from the designer.</blockquote><h4>8. Activity Editors with Intellisense support powered by Monaco Editor</h4><p>Activity Editors now use Monaco Editor for editing workflow expressions with support for JavaScript and Liquid.</p><p>This enables the use of intellisense, as seen in the following animation:</p><figure><img data-image-id="1*0yqNBzKZV-uz7V6aFA4nSQ.gif" data-width="1742" data-height="849" src="https://cdn-images-1.medium.com/max/800/1*0yqNBzKZV-uz7V6aFA4nSQ.gif"><figcaption>Elsa now comes with Intellisense support</figcaption></figure><blockquote>Small caveat: intellisense requires types and functions to be registered explicitly, which hasn’t been done for every function as of yet. This is an ongoing effort.</blockquote><h4>9. Workflow Context Support</h4><p>Elsa has a new concept called “Workflow Context”.</p><p>The idea here is that many workflows typically process some form of domain model that is persisted elsewhere (could be the same database or any other place).</p><p>Instead of having to write custom activities that load &amp; persist these models, Elsa can now do this for you automatically.</p><p>All you have to do is implement a workflow context provider class and tell your workflow the .NET type of your model.</p><p>You can then configure the workflow when the context needs to be loaded. For example, do you want to load the object once before a “burst of execution” or before each and every activity executes? Or perhaps you want to control this on a per-activity basis? All you have to do is check some boxes.</p><p>You can learn more about this feature <a href="https://elsa-workflows.github.io/elsa-core/docs/next/concepts/concepts-workflow-context" rel="noopener" target="_blank">here</a>.</p><h4>10. New activities</h4><p>Besides improving some of the existing activities, we added new ones as well, such as:</p><ul><li>Azure Service Bus activities to send and receive messages using Azure Service Bus</li><li>Rebus activities to send and receive messages using Rebus</li><li>Blob storage activities to read, write and delete blobs from a configured storage provider using <a href="https://github.com/aloneguid/storage" rel="noopener" target="_blank">Storage.Net</a></li><li><a href="https://telnyx.com/" rel="noopener" target="_blank">Telnyx</a> activities that integrate with the Telnyx APIs for telephony applications</li><li>Break activity that allows you to break out of a loop such as constructed with the For, ForEach and While activities</li><li>Switch activity that loosely models the C# <code>switch</code> construct</li><li>RunWorkflow activity, which allows you to execute a specific workflow by ID or name. You can execute a workflow in a “fire and forget” fashion or suspend workflow execution until the child workflow finishes.</li></ul><h3>What’s Fixed</h3><ul><li>Serialization issues have been fixed</li><li>Temporal activities such as Timer, Cron and StartAt now work reliably by relying on providers such as <a href="https://www.quartz-scheduler.net/" rel="noopener" target="_blank">Quartz.NET</a> and <a href="https://www.hangfire.io/" rel="noopener" target="_blank">Hangfire</a></li><li>Workflow Execution Log is now stored separate from Workflow Instance</li><li>Many other issues too numerous to list here</li></ul><p>For a full list of changes, fixes and improvements, checkout the <a href="https://github.com/elsa-workflows/elsa-core/milestone/8?closed=1" rel="noopener" target="_blank">Elsa 2.0 Milestone on GitHub</a>.</p><h3>Getting Started</h3><p>To get started with Elsa 2, make sure to checkout the <a href="https://elsa-workflows.github.io/elsa-core/docs/next/quickstarts/quickstarts-console-hello-world" rel="noopener" target="_blank">documentation website</a>.</p><p>And if you just want to take it for a quick spin, run this Docker command:</p><pre>docker run -t -i -e ELSA__SERVER__BASEURL=&#39;<a href="http://localhost:13000%27" rel="nofollow noopener" target="_blank">http://localhost:13000&#39;</a> -p 13000:80 elsaworkflows/elsa-dashboard-and-server:latest</pre><p>Once the container is running, simply go to <a href="http://localhost:13000%27" rel="nofollow noopener noopener" target="_blank">http://localhost:13000</a> and click around.</p></div><div><figure><img data-image-id="1*WxHYxAuXmr0tFmVdcFTokQ.gif" data-width="2007" data-height="1275" src="https://cdn-images-1.medium.com/max/1200/1*WxHYxAuXmr0tFmVdcFTokQ.gif"></figure></div><div><h3>Attribution</h3><p>This release was made possible thanks to many contributors who generously donated their time and energy by testing out features, reporting issues and contributing code.
