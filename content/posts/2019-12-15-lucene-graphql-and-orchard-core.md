<section><div><div><h3>Lucene, GraphQL and Orchard Core</h3><h4>A brief guide on implementing Search using Orchard Core, Lucene and GraphQL.</h4></div><div><figure><img data-image-id="1*-h6Gwv5yDhj5V_QIeInZfw.jpeg" data-width="4712" data-height="1782" data-is-featured="true" src="https://cdn-images-1.medium.com/max/2560/1*-h6Gwv5yDhj5V_QIeInZfw.jpeg"><figcaption>Photo by <a href="https://unsplash.com/@gerandeklerk?utm_source=unsplash&amp;utm_medium=referral&amp;utm_content=creditCopyText" rel="noopener" target="_blank">Geran de Klerk</a> on <a href="https://unsplash.com/s/photos/search?utm_source=unsplash&amp;utm_medium=referral&amp;utm_content=creditCopyText" rel="noopener" target="_blank">Unsplash</a></figcaption></figure></div><div><p>Recently I was asked how one would go about enabling search functionality in a front-end single-page-application (SPA) that is powered by <a href="https://www.orchardcore.net/" rel="noopener" target="_blank">Orchard Core CMS</a>.</p><p>Now, I already knew that Orchard provides search functionality using <a href="http://lucenenet.apache.org/" rel="noopener" target="_blank">Lucene.NET</a>, so I figured its Lucene module probably exposes a REST API that the SPA should be able to call. Sure enough, <a href="https://orchardcore.readthedocs.io/en/dev/docs/reference/modules/Lucene/#web-apis" rel="noopener" target="_blank">the documentation</a> confirmed this.</p><p>What I didn’t know however, is that the Lucene module integrates with GraphQL, effectively allowing one to query Lucene indices using GraphQL.</p><p>Say what?</p><p>Yeah that’s right. We can query Lucene indices using GraphQL, right out of the box. GraphQL offers great flexibility when it comes to querying any number of data sources. But the consumer needn’t know nor care about any of that. It only specifies what type it wants to query, and what fields it needs back from it. Be it content items, other APIs, or Lucene indices. All uniformly available through a consistent API with GraphQL.</p><p>Let’s see how this works!</p></div></div></section><section><div><div><h3>Enabling Lucene</h3><p>The first thing to do before we can query anything, is to make sure that the <em>Lucene</em> feature is enabled. Depending on the recipe you selected during setup, this may or may not be the case already. If you selected the <em>Blog</em> recipe, then Lucene will be enabled for you.</p><figure><img data-image-id="1*xQLk8FcKMI2zkE0nfQxj3Q.png" data-width="1755" data-height="1300" src="https://cdn-images-1.medium.com/max/800/1*xQLk8FcKMI2zkE0nfQxj3Q.png"><figcaption>Select the Blog recipe and your site will have Lucene enabled by default.</figcaption></figure><p>If you’re not sure if Lucene is enabled, go to the Admin Dashboard, then select <em>Configuration → Features</em> from the left menu, and enable the feature called <em>Lucene</em> if it’s disabled.</p><figure><img data-image-id="1*owpW-muNAY2zcpkXFDwHlg.png" data-width="1916" data-height="1209" src="https://cdn-images-1.medium.com/max/800/1*owpW-muNAY2zcpkXFDwHlg.png"><figcaption>Lucene is enabled.</figcaption></figure><h3>Setting up a Lucene Query</h3><p>To demonstrate how you can query Lucene indices, we’ll create a new Query object using the Admin Dashboard that returns all blog posts containing a user-provider search term.</p><p>On the left menu, select <em>Search → Queries → All Queries</em>, then select the <em>Add Query</em> button.</p><figure><img data-image-id="1*u5Wj-Nx1XUwnwR6FxEvivQ.png" data-width="1916" data-height="1209" src="https://cdn-images-1.medium.com/max/800/1*u5Wj-Nx1XUwnwR6FxEvivQ.png"><figcaption>Select <strong><em>Add Query</em></strong></figcaption></figure><p>A modal dialog appears, showing all available <em>Query Types</em>. Orchard Core being a modular, extensible framework, might have any number of modules installed that adds to the types of queries we can use. We are just interested in creating a <em>Lucene</em> query, so select that one.</p><figure><img data-image-id="1*Ii0u2LCASDMMCN1RcQAQAw.png" data-width="1916" data-height="1209" src="https://cdn-images-1.medium.com/max/800/1*Ii0u2LCASDMMCN1RcQAQAw.png"><figcaption>Select the Lucene query type</figcaption></figure><p>On the Lucene query editor screen, enter the following details:</p><ul><li>Name: <code>BlogPostsQuery</code></li><li>Schema: <br><code>{<br> &quot;type&quot;: &quot;ContentItem/BlogPost&quot;<br>}</code></li><li>Index: <code>Search</code></li><li>Return Content Items: <code>checked</code></li><li>Query:</li></ul><pre>{<br>  &quot;query&quot;: {<br>    &quot;bool&quot;: {<br>      &quot;must&quot;: [<br>        {<br>          &quot;match&quot;: {<br>            &quot;Content.ContentItem.FullText&quot;: &quot;{{ Term }}&quot;<br>          }<br>        },<br>        {<br>          &quot;term&quot;: {<br>            &quot;Content.ContentItem.ContentType&quot;: &quot;BlogPost&quot;<br>          }<br>        }<br>      ]<br>    }<br>  }<br>}</pre><p>The above query is a liquified (templated using <a href="https://shopify.github.io/liquid/" rel="noopener" target="_blank">Liquid</a> syntax) Lucene query that finds all content items that:</p><ol><li>Have a field called <code>Content.ContentItem.FullText</code> containing whatever string value returned by the Liquid variable <code>Term</code> (we’ll see how to provide a value for that shortly).</li><li>Have term field called <code>Content.ContentItem.ContentType</code> having the exact value of <code>BlogPost</code>.</li></ol><p>This query effectively yields all content items of type BlogPost containing a yet-to-be-defined search term that we need to specify as a parameter.</p><p>Make sure to select the <em>Save</em> button before we try out our query.</p><h3>Testing the Lucene Query</h3><p>From the left menu, select <em>Search → Queries → All Queries</em>. You’ll see the query we just created, and 3 buttons called <em>Run</em>, <em>Edit</em> and <em>Delete</em>. Select the <em>Run</em> button.</p><figure><img data-image-id="1*yaVy50EC9lNSnzCwTmZMzA.png" data-width="2159" data-height="911" src="https://cdn-images-1.medium.com/max/800/1*yaVy50EC9lNSnzCwTmZMzA.png"><figcaption>Select the <strong><em>Run</em></strong> button on the newly created <strong><em>BlogPostsQuery</em>.</strong></figcaption></figure><p>On the next screen, we see our query template, as well as a <em>Parameters</em> field. That’s where we’ll supply the <code>Term</code> parameter that we defined in the query template. As an example, we’ll try looking for all blog posts containing the word “explore” (which should yield exactly 1 result if you used the Blog recipe).</p><figure><img data-image-id="1*3VcSbByGrgajITujHxprUQ.png" data-width="1732" data-height="1658" src="https://cdn-images-1.medium.com/max/800/1*3VcSbByGrgajITujHxprUQ.png"><figcaption>Running the query with “explore” as the Term value yields exactly one matching document.</figcaption></figure><p>Now we’ve seen that that works, we’re ready to try out the query using an API call from Postman. After that, we’ll do the same, but using a GraphQL query API call.</p><h3>Lucene + GraphQL</h3><p>Finally, it’s time to query the Lucene index using GraphQL, which enables us to specify exactly what fields to be included in the response.</p><p>First, we’ll need to enable the <em>GraphQL</em> feature:</p><figure><img data-image-id="1*zqqYb2S-CA8DtwAYlCbD_A.png" data-width="2159" data-height="977" src="https://cdn-images-1.medium.com/max/800/1*zqqYb2S-CA8DtwAYlCbD_A.png"><figcaption>Enable the GraphQL feature.</figcaption></figure><p>Before we see how to invoke the GraphQL endpoint using a HTTP request, let’s first try it out using <a href="https://www.gatsbyjs.org/docs/running-queries-with-graphiql/" rel="noopener" target="_blank"><em>GraphiQL</em></a>. From the left menu, select <em>Configuration → GraphiQL</em>.</p><p>Notice that the Explorer view contains our <em>BlogPostsQuery</em> query:</p><figure><img data-image-id="1*94N3cbkzDs5zkqXp9jbhIQ.png" data-width="1073" data-height="1027" src="https://cdn-images-1.medium.com/max/800/1*94N3cbkzDs5zkqXp9jbhIQ.png"><figcaption>The GraphiQL explorer shows our BlogPostsQuery as a Type.</figcaption></figure><p>Go ahead, and select/enter the following GraphQL query:</p><pre>query MyQuery($parameters: String!) {<br>  blogPostsQuery(parameters: $parameters) {<br>    markdownBody {<br>      html<br>    }<br>    subtitle<br>    author<br>    displayText<br>  }<br>}</pre><p>Notice that I’m passing in a GraphQL variable named <code>$parameters</code>, so make sure to define that variable in the Query Variables view:</p><pre>{<br>  &quot;parameters&quot;: &quot;{\&quot;Term\&quot;: \&quot;explore\&quot;}&quot;<br>}</pre><p>When selecting the <em>Play</em> button, you should receive the following response:</p><pre>{<br>  &quot;data&quot;: {<br>    &quot;blogPostsQuery&quot;: [<br>      {<br>        &quot;markdownBody&quot;: {<br>          &quot;html&quot;: &quot;&lt;p&gt;Never in all their history have men been able truly to conceive of the world as one: a single sphere, a globe, having the qualities of a globe, a round earth in which all the directions eventually meet, in which there is no center because every point, or none, is center — an equal earth which all men occupy as equals.
