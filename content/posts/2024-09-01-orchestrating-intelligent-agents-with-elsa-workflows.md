<section><div><div><h3>Orchestrating Intelligent Agents with Elsa Workflows</h3><p>Outsource Cognitive Tasks and Deploy a Team of Intelligent Agents to Supercharge Your Business Processes</p></div><div><figure><img data-image-id="1*ZLTuS_LxTRqDzKO8WWIh1Q.png" data-width="3830" data-height="1990" data-is-featured="true" src="https://cdn-images-1.medium.com/max/1200/1*ZLTuS_LxTRqDzKO8WWIh1Q.png"></figure></div><div><h3>Introduction</h3><p>A set of new modules just landed in the preview of Elsa 3.3 called <strong>Agents</strong>.</p><p>With Agents, you can define a set of personas that are specialized in a given cognitive task. These agents can then be added to a workflow from where they will perform their work as a background task and produce output that can then feed into subsequent activities — including other agents.</p><blockquote>In this context, an agent refers to a preconfigured large language model (LLM) prompt, combined with execution settings and inputs that are provided to the LLM for processing.</blockquote><p>The ability to outsource cognitive tasks to intelligent agents enables powerful use cases. For example, if you’re automating content publishing pipelines, you could deploy a Proofreader agent that checks articles for typos, grammar errors and spelling mistakes and fix them. Another agent might create some accompanying artwork, while another agent prepares meta description + keywords for SEO optimization and another agent prepares social media outings.</p><p>The potential is virtually limitless.</p></div></div></section><section><div><div><h3>Setting Up Agents</h3><p>Let’s explore the Agents module in practice within the context of an actual Elsa Server application.</p><blockquote>In case you don’t have an Elsa Server project setup yet, please follow <a href="https://elsa-workflows.github.io/elsa-documentation/elsa-server.html" rel="noopener" target="_blank">this guide</a>.</blockquote><p>Setting up your Elsa Server with support for Agents involves these main steps:</p><ol><li>Adding references to the relevant <em>Elsa.Agents.*</em> packages.</li><li>Configuring Agents in <code>appsettings.json</code>.</li><li>Updating <code>Program.cs</code> .</li></ol><h4>Package References</h4><p>Add the following packages to your Elsa Server project:</p><pre data-code-block-mode="2" spellcheck="false" data-code-block-lang="bash"><span>dotnet add package Elsa.Agents.Core --<span>source</span> https://f.feedz.io/elsa-workflows/elsa-3/nuget/index.json<br />dotnet add package Elsa.Agents.Activities --<span>source</span> https://f.feedz.io/elsa-workflows/elsa-3/nuget/index.json</span></pre><p>These packages contain the core entities and services to define agents and interact with <a href="https://learn.microsoft.com/en-us/semantic-kernel/overview/" rel="noopener" target="_blank">Semantic Kernel</a>.</p><p>The <em>Elsa.Agents.Activities</em> package dynamically generates activities based on agents defined in the application.
