<h4>Modeling multitenancy through container isolation instead of runtime conditionals</h4><figure><img alt="" src="https://cdn-images-1.medium.com/max/1024/1*_DV5XBtHU1P9L5iYYBeC8A.jpeg" /></figure><p>Multitenancy raises a structural question: where does tenant-specific behavior live?</p><p>Different tenants may require different features or configuration. One enables comments. Another enables analytics with different limits. A third integrates with an external system that the others do not. These differences must be represented somewhere in the system.</p><p>There are two fundamentally different ways to model this:</p><ol><li><strong>Runtime variation</strong>: services inspect tenant identity and branch.</li><li><strong>Structural variation</strong>: tenants are isolated, and differences are expressed by composition.</li></ol><p>In the first model, tenant identity flows through the application as data. In the second, tenant identity determines how the system is constructed.</p><p><a href="https://www.cshells.io/">CShells</a> implements the second model.</p><p>Each tenant is modeled as a shell — an isolated execution context with its own IServiceProvider, its own IConfiguration, and its own set of enabled features. Features register services and optionally expose endpoints. Shells enable features through configuration.</p><p>To demonstrate how this works, this article builds a small blog platform:</p><ul><li>Default: Core, Posts</li><li>Acme: Core, Posts, Comments</li><li>Contoso: Core, Posts, Comments, Analytics under /contoso/*</li></ul><p>Behavioral differences are expressed by enabling or disabling features per shell.</p><h3>Architectural Model</h3><h3>Shell</h3><p>A shell represents an application boundary inside a larger host process.</p><p>Each shell has:</p><ul><li>Its own IServiceProvider</li><li>Its own IConfiguration</li><li>A defined set of enabled features</li></ul><p>When the application starts, CShells builds a separate dependency injection container for each shell. Service registrations are therefore isolated. A service registered in one shell does not exist in another.</p><p>From the perspective of application code, there is no concept of “current tenant ID” being passed around. Code resolves services from the container, and the container itself encodes tenant differences.</p><h3>Feature</h3><p>A feature is a modular unit of behavior. It implements:</p><ul><li>IShellFeature (service registration)</li><li>IWebShellFeature (service registration + endpoint mapping)</li></ul><p>Features are discovered via [ShellFeature] and may declare dependencies using DependsOn.</p><p>A feature does not know which tenants use it. It only defines services and endpoints. Whether it is active depends entirely on shell configuration.</p><p>This separation allows features to be composed declaratively rather than conditionally.</p><h3>Structure</h3><p>Install packages:</p><pre>dotnet add package CShells<br>dotnet add package CShells.AspNetCore</pre><p>Typical layout:</p><pre>YourSolution/<br>├── src/<br>│   ├── YourApp/<br>│   └── YourApp.Features/</pre><p>The feature project references CShells.AspNetCore.Abstractions. The host references CShells and CShells.AspNetCore.</p><p>Feature assemblies remain independent from host concerns.</p><h3>Core Feature</h3><p>Registers tenant metadata and exposes a root endpoint.</p><pre>[ShellFeature(&quot;Core&quot;)]<br>public class CoreFeature : IWebShellFeature<br>{<br>    public void ConfigureServices(IServiceCollection services)<br>    {<br>        services.AddSingleton&lt;ITenantInfo&gt;(sp =&gt;<br>        {<br>            var settings = sp.GetRequiredService&lt;ShellSettings&gt;();<br>            var config   = sp.GetRequiredService&lt;IConfiguration&gt;();<br><br>            return new TenantInfo<br>            {<br>                TenantId   = settings.Id.ToString(),<br>                TenantName = settings.Id.Name,<br>                Plan       = config[&quot;Plan&quot;] ?? &quot;Free&quot;<br>            };<br>        });<br>    }<br><br>    public void MapEndpoints(IEndpointRouteBuilder endpoints, IHostEnvironment? environment)<br>    {<br>        endpoints.MapGet(&quot;/&quot;, (ITenantInfo tenant, ShellSettings settings) =&gt;<br>            Results.Ok(new<br>            {<br>                tenant   = tenant.TenantName,<br>                plan     = tenant.Plan,<br>                features = settings.EnabledFeatures<br>            }));<br>    }<br>}</pre><p>Each shell resolves its own ShellSettings and configuration values.</p><h3>Posts</h3><pre>[ShellFeature(&quot;Posts&quot;, DependsOn = [&quot;Core&quot;])]<br>public class PostsFeature : IWebShellFeature<br>{<br>    public void ConfigureServices(IServiceCollection services)<br>    {<br>        services.AddSingleton&lt;IPostRepository, InMemoryPostRepository&gt;();<br>        services.AddSingleton&lt;IShellActivatedHandler, SeedPostsHandler&gt;();<br>    }<br><br>    public void MapEndpoints(IEndpointRouteBuilder endpoints, IHostEnvironment? environment)<br>    {<br>        endpoints.MapGet(&quot;/posts&quot;,<br>            (IPostRepository repo, ITenantInfo tenant) =&gt;<br>                Results.Ok(new { tenant = tenant.TenantName, posts = repo.GetAll() }));<br>        <br>        endpoints.MapPost(&quot;/posts&quot;,<br>            async (HttpContext ctx, IPostRepository repo) =&gt;<br>            {<br>                var req = await ctx.Request.ReadFromJsonAsync&lt;CreatePostRequest&gt;();<br>                if (req is null) return Results.BadRequest();<br>                var post = repo.Add(req.Title, req.Body, req.Author);<br>                return Results.Created($&quot;posts/{post.Id}&quot;, post);<br>            });<br>    }<br>}</pre><p>Each shell receives its own repository instance. No tenant branching is required.</p><pre>public class SeedPostsHandler(IPostRepository repo, ITenantInfo tenant)<br>    : IShellActivatedHandler<br>{<br>    public Task OnActivatedAsync(CancellationToken _)<br>    {<br>        repo.Add($&quot;Welcome to {tenant.TenantName}&quot;,<br>                 $&quot;Plan: {tenant.Plan}&quot;,<br>                 &quot;System&quot;);<br>        return Task.CompletedTask;<br>    }<br>}</pre><h3>Comments</h3><pre>[ShellFeature(&quot;Comments&quot;, DependsOn = [&quot;Posts&quot;])]<br>public class CommentsFeature : IWebShellFeature<br>{<br>    public void ConfigureServices(IServiceCollection services)<br>        =&gt; services.AddSingleton&lt;ICommentRepository, InMemoryCommentRepository&gt;();<br><br>    public void MapEndpoints(IEndpointRouteBuilder endpoints, IHostEnvironment? environment)<br>    {<br>        endpoints.MapGet(&quot;/posts/{id:int}/comments&quot;,<br>            (int id, ICommentRepository repo) =&gt;<br>                Results.Ok(repo.GetByPostId(id)));<br>    }<br>}</pre><p>If a shell does not enable Comments, neither its services nor its routes exist.</p><h3>Analytics</h3><pre>[ShellFeature(&quot;Analytics&quot;, DependsOn = [&quot;Posts&quot;])]<br>public class AnalyticsFeature : IWebShellFeature<br>{<br>    public void ConfigureServices(IServiceCollection services)<br>    {<br>        services.AddOptions&lt;AnalyticsOptions&gt;()<br>            .Configure&lt;IConfiguration&gt;((opts, config) =&gt;<br>                config.GetSection(&quot;Analytics&quot;).Bind(opts));<br><br>        services.AddSingleton&lt;IAnalyticsService, InMemoryAnalyticsService&gt;();<br>    }<br><br>    public void MapEndpoints(IEndpointRouteBuilder endpoints, IHostEnvironment? environment)<br>    {<br>        endpoints.MapGet(&quot;/analytics&quot;,<br>            (IAnalyticsService analytics,<br>             IOptions&lt;AnalyticsOptions&gt; options) =&gt;<br>            {<br>                var top = analytics.GetViewCounts()<br>                    .OrderByDescending(kv =&gt; kv.Value)<br>                    .Take(options.Value.TopPostsCount);<br>                return Results.Ok(top);<br>            });<br>    }<br>}</pre><p>Options are bound per shell from that shell’s configuration.</p><h3>Configuration</h3><p>Shells are defined under the CShells section:</p><pre>{<br>  &quot;CShells&quot;: {<br>    &quot;Shells&quot;: [<br>      {<br>        &quot;Name&quot;: &quot;Default&quot;,<br>        &quot;Features&quot;: [ &quot;Core&quot;, &quot;Posts&quot; ],<br>        &quot;Configuration&quot;: {<br>          &quot;WebRouting:Path&quot;: &quot;&quot;,<br>          &quot;Plan&quot;: &quot;Free&quot;<br>        }<br>      },<br>      {<br>        &quot;Name&quot;: &quot;Acme&quot;,<br>        &quot;Features&quot;: [ &quot;Core&quot;, &quot;Posts&quot;, &quot;Comments&quot; ],<br>        &quot;Configuration&quot;: {<br>          &quot;WebRouting:Path&quot;: &quot;acme&quot;,<br>          &quot;Plan&quot;: &quot;Pro&quot;<br>        }<br>      },<br>      {<br>        &quot;Name&quot;: &quot;Contoso&quot;,<br>        &quot;Features&quot;: [<br>          &quot;Core&quot;,<br>          &quot;Posts&quot;,<br>          &quot;Comments&quot;,<br>          { &quot;Name&quot;: &quot;Analytics&quot;, &quot;TopPostsCount&quot;: 10 }<br>        ],<br>        &quot;Configuration&quot;: {<br>          &quot;WebRouting:Path&quot;: &quot;contoso&quot;,<br>          &quot;Plan&quot;: &quot;Enterprise&quot;<br>        }<br>      }<br>    ]<br>  }<br>}</pre><p>Each shell receives its own IConfiguration instance derived from this configuration data. Feature objects defined inline (e.g., Analytics) contribute their values to that shell’s configuration.</p><h3>Host</h3><pre>var builder = WebApplication.CreateBuilder(args);<br><br>builder.AddShells([typeof(CoreFeature)]);<br>var app = builder.Build();<br>app.UseRouting();<br>app.MapShells();<br>app.Run();</pre><p>AddShells discovers features and loads shells. MapShells resolves the active shell per request and mounts endpoints accordingly.</p><h3>Summary</h3><p>CShells models multitenancy as structural isolation:</p><ul><li>Each shell has its own container and configuration.</li><li>Services and endpoints exist only if their feature is enabled.</li><li>Differences are expressed declaratively in configuration.</li><li>Tenant identity does not flow through domain logic.</li></ul><p>The full sample is available under samples/CShells.Workbench.</p>
