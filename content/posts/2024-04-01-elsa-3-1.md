<h4>Elevating Reliability, Usability, and Performance</h4><figure><img alt="" src="https://cdn-images-1.medium.com/max/800/1*yxru9HF3heSCxIo9K_gu-Q.png" /></figure><p>We are excited to announce the arrival of <a href="https://github.com/elsa-workflows/elsa-core/releases/tag/3.1.0">ELSA 3.1</a>, a version that stands as a testament to our commitment towards delivering an even more stable, reliable, and user-friendly workflow automation platform.</p><p>In case you are unfamiliar with Elsa Workflows:</p><blockquote>Elsa Workflows is a powerful and flexible execution engine, encapsulated as a set of open-source .NET libraries designed to infuse .NET applications with workflow capabilities. With Elsa, developers can weave logic directly into their systems, enhancing functionality and automation and align seamlessly with the application’s core functionality.</blockquote><blockquote>Visit the <a href="https://v3.elsaworkflows.io/">ELSA documentation website</a> for more information.</blockquote><h3>Release Overview</h3><p>In this release, we’ve made significant strides in refining the features and functionalities of ELSA, focusing on enhancing system stability, reliability and performance. Here are the highlights:</p><h4>New &amp; Improved Features</h4><ul><li><strong>Bulk Dispatch Workflows Activity</strong>: Introducing an activity to dispatch workflows based on a list of objects, streamlining process automation.</li></ul><figure><img alt="" src="https://cdn-images-1.medium.com/max/1024/1*5pjRxMdAEz72y-F3bwGOKQ.png" /><figcaption>The Bulk Dispatch Workflows Activity</figcaption></figure><ul><li><strong>Parallel For Each Activity</strong>: Enables concurrent processing, optimizing workflow execution time. The key difference between For Each and Parallel For Each is that the former will schedule its Body activity one by one, after each completion of the current iteration, while the latter will schedule its Body activity immediately for all items in the list. Although the scheduled activities are still processed sequentially, these activities can run in the background, which then enables true parallel execution. This approach is discussed in more detail in my article about <a href="https://medium.com/@sipkeschoorstra/parallel-task-orchestration-with-elsa-3-0-e387f863655f">Parallel Task Orchestration with ELSA 3.0</a>.</li><li><strong>Workflow Cancellation</strong>: Empowering developers with the ability to programmatically cancel workflows, adding flexibility to process management.<br>Here’s a sample curl to cancel a given workflow instance:<br>curl — location — request POST ‘<a href="https://localhost:5001/elsa/api/cancel/workflow-instances/b355d0008d595725&#39;">https://localhost:5001/elsa/api/cancel/workflow-instances/b355d0008d595725&#39;</a> \<br> — header ‘Authorization: ApiKey {api-key}’<br>We have also updated the UI to allow users to cancel workflows in various ways: Bulk Cancel, cancel individual workflow instances, and cancel all workflow instances of a given workflow definition.</li></ul><figure><img alt="" src="https://cdn-images-1.medium.com/max/1024/1*Ac5Bn-1XpUEIE5UJo18JgQ.png" /><figcaption>Bulk Cancel workflows</figcaption></figure><figure><img alt="" src="https://cdn-images-1.medium.com/max/1024/1*vGHvJL95GEwXDQdYiicMMw.png" /><figcaption>Cancel all workflow instances of a given workflow definition</figcaption></figure><ul><li><strong>Workflow Definition &amp; Instance Search</strong>: Updated with search capabilities, making navigation and management more intuitive.</li></ul><figure><img alt="" src="https://cdn-images-1.medium.com/max/1024/1*vYv6e_p4syrdqe9gPJBXZQ.png" /><figcaption>A new search field was added to the Workflow Definitions list view</figcaption></figure><figure><img alt="" src="https://cdn-images-1.medium.com/max/1024/1*qEJxZ_qtHENg3FDnDFBRlg.png" /><figcaption>The Workflow Instances UI received even more search capabilities</figcaption></figure><ul><li><strong>Extended JavaScript Support</strong>: Array functions in JavaScript now seamlessly work with .NET lists, enhancing interoperability.</li><li><strong>Workflow Input</strong>: New inputs augment rather than overwrite existing ones, preserving data integrity.</li></ul><h4>Performance and Efficiency Improvements</h4><ul><li><strong>Compressed Workflow Instance Data</strong>: An optional setting to store Workflow Instance Data in a compressed format significantly reduces storage requirements.</li></ul><pre>services.AddElsa(elsa =&gt;<br>{<br>  elsa.UseManagement(management =&gt;<br>  {<br>    management.SetCompressionAlgorithm(nameof(Zstd));<br>  }<br>});</pre><ul><li><strong>Optimized Activity Execution Contexts</strong>: We no longer persist completed Activity Execution Contexts, which cuts down on data storage and contributes to overall efficiency.</li></ul><h4>Reliability and Bug Fixes</h4><ul><li><strong>Memory Leak Resolution</strong>: What can we say; fixing this critical issue was a significant improvement in the platform’s stability.</li><li><strong>Serialization and Deserialization Improvements</strong>: Fixes in these areas ensure better serialization support for a variety of data types.</li></ul><h4>Developer Productivity</h4><ul><li><strong>Cancellation and Alterations API</strong>: Enhanced capabilities in canceling workflows and activities offer more control over running workflows.</li><li><strong>.NET 8 SDK Support</strong>: Keeping pace with the latest .NET developments, we ensure ELSA remains on the cutting edge.</li><li><strong>Central Package Management</strong>: Simplifies NuGet package management across the vast array of ELSA module projects, enhancing project maintainability.</li><li><a href="https://www.jetbrains.com/writerside/"><strong>Writerside</strong></a><strong> Implementation</strong>: An initiative to improve the documentation experience, <a href="https://elsa-workflows.github.io/elsa-documentation/introduction.html">currently in preview</a>.</li></ul><h3>Community Shout Out</h3><p>As we celebrate the launch of ELSA 3.1, we want to extend a heartfelt thank you to our vibrant community. Thank you for your contributions, for believing in the vision of ELSA, and for being an integral part of our community. Here’s to many more collaborations and successes in the future!</p><h3>Looking Ahead</h3><p>Our vision for the next release, ELSA 3.2, is firmly anchored in enhancing the platform’s core capabilities. Our commitment is to deliver an even more robust, user-friendly, and secure environment for all users. Here’s a glimpse into the key areas of focus:</p><h4>Enhanced Performance &amp; Scalability with Proto.Actor</h4><p>To further address and improve performance bottlenecks, we’re looking to design a robust, distributed computing layer using <a href="https://proto.actor/">Proto.Actor</a>. This strategic move, guided by expert insights, is aimed at optimizing distributed workflow processing, enhancing ELSA’s scalability and reliability in significant ways.</p><h4>Role-Based Access Control (RBAC)</h4><p>Advancing security with RBAC: Security remains a top priority, and with ELSA 3.2, we’re taking a significant step forward by introducing a lightweight Role-Based Access Control system. This feature will provide finer control over who can do what within ELSA, ensuring users have access only to the features and data relevant to their roles. The implementation of RBAC is fundamental to supporting secure and controlled access across ELSA’s functionalities, offering peace of mind to teams and enterprises alike.</p><h4><a href="https://github.com/elsa-workflows/elsa-core/issues/4833">Workflow Execution Recovery</a></h4><p>At the heart of any robust workflow engine lies the capability to recover and resume operations seamlessly after unexpected interruptions. Recognizing the critical importance of this feature, we are excited to announce our focus on Workflow Execution Recovery for our upcoming milestone.</p><h4><a href="https://github.com/elsa-workflows/elsa-core/issues/764">Multitenancy</a></h4><p>In collaboration with our vibrant community, we’re introducing Multitenancy in our next release.</p><p>In this first iteration, we will support the <em>shared database, shared schema multitenancy</em> type. In this type, all tenants share the same database and schema. Data for all tenants are stored in the same tables, and a “tenant ID” column is used to distinguish between data owned by different tenants. This approach maximizes resource sharing and simplifies the architecture but requires robust data access controls to prevent data leaks between tenants.</p><h4>Usability Enhancements</h4><p>Recognizing the critical role of user experience, our next release prioritizes resolving longstanding usability challenges. From refining the UI for intuitive navigation to implementing lazy loading for efficiency, and correcting timezone inconsistencies, these enhancements are designed to make ELSA more accessible and enjoyable to use.</p><h4>Ongoing Commitment to Code Health</h4><p>A steadfast dedication to the platform’s health underpins our next efforts. ELSA 3.2 will see a concerted push to reduce technical debt and address critical bugs, ensuring the platform remains clean, efficient, and future-proof. This commitment not only enhances stability but also secures the foundation for introducing new features.</p><h3>Summary</h3><p>ELSA 3.1 is out and available on NuGet, marking a significant step forward in our mission to provide a top-tier workflow automation platform. This latest release is packed with improvements across the board, reflecting our unwavering commitment to reliability, usability, and performance.</p><p>With the release of ELSA 3.1 out of the door, we already set our sights on the horizon with the upcoming 3.2 milestone, which is slated for June 2024. Key among these are the introduction of multitenancy and advanced security through Role-Based Access Control (RBAC), coupled with significant performance upgrades powered by the integration of Proto.Actor for distributed computing.</p><p>The journey of ELSA is one of continuous innovation and dedication to excellence. With each release, we aim to exceed your expectations, pushing the boundaries of what is possible in workflow automation. The road ahead is filled with potential, and we invite you to join us on this exciting journey as we explore new heights with ELSA 3.2 and beyond.</p><h3>Resources</h3><ul><li><a href="https://github.com/elsa-workflows/elsa-core/releases/tag/3.1.0">3.1 Release Notes</a></li><li><a href="https://medium.com/@sipkeschoorstra/elsa-3-0-2b341e7cbfa7">3.0 Introduction Blog Post</a></li><li><a href="https://v3.elsaworkflows.io/">Documentation</a></li><li><a href="https://github.com/elsa-workflows/elsa-core">Source Code</a></li></ul>
